Linux (PipeWire)

Create a network tunnel that forwards local audio to the SinkSonic Pi over TCP. The script runs as a persistent systemd user service and auto-reconnects.

One-liner Setup

Loading...

Or manually

Create the script:

mkdir -p ~/.local/bin curl -o ~/.local/bin/sinksonic-sink.sh http://sinksonic.local/api/setup/linux chmod +x ~/.local/bin/sinksonic-sink.sh

Create the systemd service:

mkdir -p ~/.config/systemd/user cat > ~/.config/systemd/user/sinksonic-sink.service << 'EOF' [Unit] Description=SinkSonic — network audio tunnel to Pi After=pipewire-pulse.service network-online.target Wants=pipewire-pulse.service [Service] Type=simple ExecStart=%h/.local/bin/sinksonic-sink.sh Restart=on-failure RestartSec=10 [Install] WantedBy=default.target EOF systemctl --user daemon-reload systemctl --user enable --now sinksonic-sink.service

Verify

Select "SinkSonic" in your sound settings, or test with:

pactl list sinks short | grep sinksonic # Look for sinksonic-tunnel in the output # Test audio PULSE_SINK=sinksonic-tunnel speaker-test -c 2 -l 1 -t sine

macOS

macOS does not natively support PulseAudio. You need to install additional software.

Option 1: PulseAudio (Homebrew)

brew install pulseaudio brew services start pulseaudio # Load the tunnel sink pactl load-module module-tunnel-sink \ server=tcp:sinksonic.local:4713 \ sink_name=sinksonic-tunnel \ sink_properties=device.description=SinkSonic # Set as default pactl set-default-sink sinksonic-tunnel

Option 2: OBS + PipeWire (via MacPorts)

Route application audio through OBS with the PipeWire plugin to the SinkSonic network sink.

Test audio

# Use audacity or any media player that can select an audio output # Select "SinkSonic" as the output device

Windows

Windows does not natively support PulseAudio. Use one of these approaches.

Option 1: PulseAudio for Windows

# Download PulseAudio for Windows: # https://www.freedesktop.org/wiki/Software/PulseAudio/Ports/Win32/ # Install and run pulseaudio.exe, then in a command prompt: pactl load-module module-tunnel-sink \ server=tcp:sinksonic.local:4713 \ sink_name=sinksonic-tunnel \ sink_properties=device.description=SinkSonic

Option 2: WSL2 with PipeWire

Run the Linux setup inside WSL2 and use mDNS forwarding to reach sinksonic.local.

# In WSL2, follow the Linux setup instructions above # Note: you may need to forward mDNS: echo "192.168.178.160 sinksonic.local" | sudo tee -a /etc/hosts

Option 3: OBS with audio routing

Use Voicemeeter to route system audio to a virtual cable, then OBS with the PulseAudio plugin to stream to SinkSonic.

Android

Android audio routing to network sinks requires third-party apps.

Option 1: AudioRelay (recommended)

AudioRelay streams Android audio to a server on your PC. The server then forwards to SinkSonic via the Linux desktop tunnel.

Option 2: SoundWire

SoundWire streams from Android to a Windows/Linux server. Works together with the desktop tunnel.

# On the desktop (Linux), after creating the SinkSonic tunnel: # Install the SoundWire server, configure it to output to SinkSonic # Then connect your Android device to the SoundWire server

Option 3: Snapcast

Snapcast is an open-source multi-room audio system. Run the Snapserver on the Pi and connect Android via the Snapcast app.

iOS

iOS severely restricts audio routing. AirPlay is the most practical option.

Option 1: AirPlay (if enabled on the Pi)

If Shairport Sync is running on the Pi:

1. Open Control Center on iOS 2. Tap the AirPlay icon 3. Select "SinkSonic" from the list 4. All device audio now streams to the Pi

Option 2: OBS + Camera app

Use the OBS Camera app to capture iOS audio and forward via network to a server that feeds SinkSonic.

Option 3: RTP Middleware

Use a third-party RTP streaming app that can forward to a configurable network endpoint.

# Pi-side: netcat can receive raw PCM # Not recommended for daily use — AirPlay is far better nc -l -p 5000 | pacat --rate=44100 --channels=2 --format=s16le
